#+title: Cabinet User Guide

* What Cabinet Is

Cabinet is an Org-backed local knowledge OS for AI-team work inside Emacs. A
Cabinet root is a directory with =index.org= plus agent, job, conversation,
task, app, import, and relationship records stored as Org files.

The main command is =M-x ogent-cabinet-home=. It opens a single operational UI
with metadata, health counts, navigation, recent activity, and work needing
attention. Failed work is shown in Cabinet Home and on the task board.

* Durable Layout

#+begin_example
cabinet-root/
  index.org
  imports/
  .agents/
    editor/
      persona.org
      inbox.org
      schedule.org
      jobs/
        weekly-review.org
      memory/
        context.org
        decisions.org
        learnings.org
      sessions/
        20260504T090000-weekly-review.org
#+end_example

Org is the durable source of truth. Commands edit property drawers and body
text in those Org files. Manual edits remain part of the record.

* First 10 Minutes

1. Create a Cabinet:

   #+begin_src emacs-lisp
   (ogent-cabinet-scaffold "~/cabinet/acme" "Acme Cabinet" :kind "root")
   #+end_src

2. Open the operating surface:

   #+begin_example
   M-x ogent-cabinet-home
   #+end_example

3. Create an agent:

   #+begin_example
   M-x ogent-cabinet-create-agent
   #+end_example

   Fill in name, slug, role, provider, model, workspace, tags, and persona.

4. Add a recurring job:

   #+begin_example
   M-x ogent-cabinet-create-job
   #+end_example

   The job is written under =.agents/<slug>/jobs/<job-id>.org=.

5. Run the job:

   Open =M-x ogent-cabinet-jobs=, move to the job, press =R=.

6. Inspect the transcript:

   Open =M-x ogent-cabinet-conversations= and press =RET= on the new run.
   The detail buffer shows metadata, prompt, output, tool blocks, errors,
   app links, duration, and source Org.

7. Search the Cabinet:

   Use =M-x ogent-cabinet-search=. Empty queries show an empty state. Search
   can filter by kind, agent, status, tag, and archive state from Lisp callers.

8. Open generated apps:

   Use =M-x ogent-cabinet-apps=. Each =index.html= below the Cabinet root is
   listed with label, path, modified time, and inferred owner when possible.

9. Check the task board:

   Use =M-x ogent-cabinet-tasks=. Lanes are Inbox, Needs Reply, Running,
   Scheduled, Just Finished, Stale, and Archive.

10. Inspect relationships:

    Use =M-x ogent-cabinet-status=. The graph includes cabinet, agent, job,
    session, app, issue-link, and Gas Town hook nodes when available. Press
    =m= for the action menu or =?= for help. Agent and job rows advertise the
    fastest edit/run actions inline.

* Importing Existing Artifacts

=ogent-cabinet-import-artifacts= imports =.md=, =.html=, and =.txt= files from
a source folder into =imports/*.org= records. Original files are kept by
default. Markdown headings are converted to Org headings. HTML is preserved in
an Org source block.

Migration limits:

- Import is conservative and does not delete originals.
- Ambiguous metadata becomes a normal import record.
- Agent and job creation from imported metadata is intentionally narrow; use
  =ogent-cabinet-create-agent= and =ogent-cabinet-create-job= when the source
  does not identify a clear agent or job.
- Imported files become Cabinet Org records, but external links and rich HTML
  behavior may need manual cleanup.

* Doom Example

#+begin_src emacs-lisp
(use-package! ogent
  :defer t
  :commands (ogent-mode
             ogent-cabinet-home
             ogent-cabinet-status
             ogent-cabinet-agents
             ogent-cabinet-agent
             ogent-cabinet-tasks
             ogent-cabinet-conversations
             ogent-cabinet-search
             ogent-cabinet-apps
             ogent-cabinet-create-agent
             ogent-cabinet-create-job)
  :init
  (setq ogent-enable-doom-bindings t
        ogent-doom-prefix "o")
  :config
  (ogent-setup-doom-bindings)
  (ogent-global-mode 1))
#+end_src
