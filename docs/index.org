* What this is

ogent is an Emacs package that brings LLMs into Org-mode as composable
Unix-style tools rather than chatbots: structured context goes in via
=@handle= references and pinned files, structured Org responses come back,
and every tool call the model makes is gated by a declarative
effects/approval policy. It is built on top of [[https://github.com/karthink/gptel][gptel]] for provider
transport, and adds a multi-agent layer (the "Armory") that runs external
coding CLIs (codex, claude, gemini, ...) against an Org-file-backed agent OS.
Source lives in =lisp/= (+ =lisp/ui/=), tests in =test/= (ERT via =make
test=), specs in =specs/=, and user docs here in =docs/=.

* Reading paths

** I have 10 minutes

1. Read the [[file:../README.org][README]] — the philosophy ("LLMs as Unix tools") and quickstart.
2. Skim the *Mental model* and *The walkthrough* sections of
   [[file:how-it-works.org][how-it-works.org]] — one real request traced end to end.
3. Keep [[file:glossary.org][glossary.org]] open for any unfamiliar term.

** I have an hour

1. The 10-minute path above.
2. [[file:getting-started.md][getting-started.md]] — install it and run =M-x ogent-onboard=, then
   =M-x ogent-prompt-dispatch= in an Org buffer.
3. [[file:how-it-works.org][how-it-works.org]] in full reading order: Mental model → Walkthrough →
   the module catalog for the subsystems you care about → Invariants →
   Failure modes → No magic.
4. [[file:armory.org][armory.org]] if you want the multi-agent layer, [[file:tools.org][tools.org]] if you want
   the tool system.
5. [[file:../run.org][run.org]] — the literate runbook: build, test, lint, sandbox, demo.

** I need to change code

1. Read the *No magic* and *Invariants* sections of
   [[file:how-it-works.org][how-it-works.org]] first — the macro layers, advice, load order, and the
   safety rules you must not break (every invariant links to the enforcing
   line).
2. Find your module's subheading in the how-it-works catalog; each one has
   Role/Design/Intuition/Trace plus exact-line links to its major defuns,
   defcustoms, and state.
3. [[file:../run.org][run.org]] — =make test= must stay green and byte-compile warnings gate
   merges; the runbook covers the sandbox and the stale-=.elc= recompile
   trap.
4. [[file:debugging.org][debugging.org]] — Edebug, =ogent-debug-mode=, and tool-call forensics.
5. [[file:../CONTRIBUTING.md][CONTRIBUTING.md]] and the issue tracker conventions in
   [[file:../AGENTS.md][AGENTS.md]] (br/beads workflow).
