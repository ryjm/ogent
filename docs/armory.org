#+title: Armory User Guide

* What Armory Is

Armory is an Org-backed local knowledge OS for AI-team work inside Emacs. A
Armory root is a directory with =index.org= plus agent, job, conversation,
task, app, import, and relationship records stored as Org files.

The main command is =M-x ogent-armory-home=. It opens a single operational UI
with metadata, health counts, navigation, recent activity, and work needing
attention. Failed work is shown in Armory Home and on the task board.

* Durable Layout

#+begin_example
armory-root/
  index.org
  imports/
  .agents/
    editor/
      persona.org
      inbox.org
      schedule.org
      jobs/
        weekly-review.org
      memory/
        context.org
        decisions.org
        learnings.org
      sessions/
        20260504T090000-weekly-review.org
#+end_example

Org is the durable source of truth. Commands edit property drawers and body
text in those Org files. Manual edits remain part of the record.

* First 10 Minutes

1. Create a Armory:

   #+begin_src emacs-lisp
   (ogent-armory-scaffold "~/armory/acme" "Acme Armory" :kind "root")
   #+end_src

2. Open the operating surface:

   #+begin_example
   M-x ogent-armory-home
   #+end_example

3. Create an agent:

   #+begin_example
   M-x ogent-armory-create-agent
   #+end_example

   Fill in name, slug, role, provider, model, workspace, tags, and persona.
   Persona files also support Armory identity fields for display name, icon,
   color, avatar, department, type, dispatch permission, adapter, effort,
   runtime mode, budget, focus paths, goals, channels, selected skills,
   recommended skills, setup status, and heartbeat state.

4. Add a recurring job:

   #+begin_example
   M-x ogent-armory-create-job
   #+end_example

   The job is written under =.agents/<slug>/jobs/<job-id>.org=.

5. Run the job:

   Open =M-x ogent-armory-jobs=, move to the job, press =C-c r=.

6. Compose an ad hoc run:

   Use =M-x ogent-armory-compose= for a minibuffer prompt, or
   =M-x ogent-armory-compose-buffer= for an Org composer buffer. Mentions are
   resolved before the provider runs and copied into the canonical conversation
   metadata.

   #+begin_example
   @agent:cto
   @page:notes/plan.org
   @skill:review
   @job:weekly-review
   @conversation:20260506T100000-run
   #+end_example

   Attach files from the composer buffer with =C-c C-a=. Staged attachments are
   copied into =.agents/.conversations/<id>/attachments/= when the run starts.

7. Inspect the transcript:

   Open =M-x ogent-armory-conversations= and press =RET= on the new run.
   The detail buffer shows actions, metadata, runtime state, turns, prompt,
   output, tool blocks, artifacts, event logs, errors, duration, and source Org.

8. Search the Armory:

   Use =M-x ogent-armory-search=. Empty queries show an empty state. Search
   can filter by kind, agent, status, tag, and archive state from Lisp callers.

9. Browse durable data:

   Use =M-x ogent-armory-data=. The data browser lists pages, files, apps, and
   folders under the Armory root. Page commands create, rename, move, delete,
   and export Org records while preserving the source property drawer and body.

10. Open the command palette:

    Use =M-x ogent-armory-command-palette=. The palette ranks commands, pages,
    agents, jobs, conversations, apps, paths, and page text through a persisted
    derived index at =.armory-state/search.el=. The minibuffer opens directly
    on completion candidates, annotated by record kind and path.

11. Open generated apps:

    Use =M-x ogent-armory-apps=. Each =index.html= below the Armory root is
    listed with label, path, modified time, and inferred owner when possible.
    HTML artifacts open through the shared Armory viewer dispatch.

12. Check git state:

    Use =M-x ogent-armory-git-status=. The status buffer lists dirty paths and
    can commit, diff, log, restore, and pull through Magit when present or plain
    git fallbacks.

13. Configure the Armory:

    Use =M-x ogent-armory-settings= for profile, provider, runtime, skills,
    storage, integration, notification, appearance, git, automation, and about
    settings. Settings are stored in =.armory-state/settings.org= and can be
    exported or imported as Org.

14. Import, demo, and back up:

    Use =M-x ogent-armory-onboard= for first-run setup, storage defaults, and
    an initial team. After choosing the default provider, model completion is
    populated from that provider adapter; adapters with a native model-listing
    command query the CLI at prompt time. Use =M-x ogent-armory-registry-import=
    to scaffold a Armory from a JSON template manifest. Use
    =M-x ogent-armory-demo= to create a small example Armory. Use
    =M-x ogent-armory-backup= to copy durable Org state while skipping git
    metadata, process state, derived search cache, and previous backups.

15. Check the task board:

    Use =M-x ogent-armory-tasks=. Lanes are Inbox, Needs Reply, Running,
    Just Finished, and Archive. Scheduled and stale states appear as item
    metadata. Press =C-c b= for board, =C-c l= for list, and =C-c S= for schedule view.

16. Inspect relationships:

    Use =M-x ogent-armory-status=. The graph includes armory, agent, job,
    session, app, issue-link, and Gas Town hook nodes when available. Press
    =C-c m= for the action menu or =C-c ?= for help. Agent and job rows advertise the
    fastest edit/run actions inline.

17. Check provider adapters:

    Use =M-x ogent-armory-providers= to inspect registered runtime adapters,
    executable availability, supported modes, and resume support. These adapter
    registrations also drive provider-aware model completion during onboarding.
    Use =M-x ogent-armory-provider-verify= for a focused executable check.

18. Browse skills:

    Use =M-x ogent-armory-skills= to inspect the skill catalog. The catalog
    reads skill metadata from =.agents/skills/=, =skills/=, =.codex/skills/=,
    =$CODEX_HOME/skills=, and =~/.agents/skills= on demand. Refer to a skill in
    a prompt with =@skill:<key>= or pass the key through the composer.

19. Review departments and approvals:

    Use =M-x ogent-armory-org-chart= for a department tree grouped by lead and
    scope. Use =M-x ogent-armory-actions= on a conversation with pending lead
    proposals to approve, reject, or dispatch actions.

20. Open the Armory schedule:

    Use =M-x ogent-armory-schedule= for day, week, and month schedule views.
    Jobs with =OGENT_CRON=, agent heartbeats with cron heartbeats, one-shot
    jobs with =OGENT_RUN_AFTER=, and scheduled conversations all appear in one
    event stream. Past slots without a matching =OGENT_SCHEDULED_KEY= show as
    missed and can be run from the schedule buffer.

21. Bridge into Org agenda:

    Use =M-x ogent-armory-agenda= to open =org-agenda= over the Armory root,
    visible child Armories, personas, inboxes, schedules, jobs, and memory Org
    records. Job files remain normal Org =TODO= entries, and run-after tasks
    get an active =SCHEDULED= timestamp.

* Importing Existing Artifacts

=ogent-armory-import-artifacts= imports =.md=, =.html=, and =.txt= files from
a source folder into =imports/*.org= records. Original files are kept by
default. Markdown headings are converted to Org headings. HTML is preserved in
an Org source block.

Migration limits:

- Import is conservative and does not delete originals.
- Ambiguous metadata becomes a normal import record.
- Agent and job creation from imported metadata is intentionally narrow; use
  =ogent-armory-create-agent= and =ogent-armory-create-job= when the source
  does not identify a clear agent or job.
- Imported files become Armory Org records, but external links and rich HTML
  behavior may need manual cleanup.
