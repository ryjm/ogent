#+title: ogent Style Guide
#+startup: overview

This document records the Emacs Lisp conventions the ogent codebase actually
follows.  Every rule cites a live example as =file:line= so the convention can
be checked against working code (citations verified 2026-06-11; treat them as
pointers, not pinned offsets).  It replaces a verbatim import of the Emacs
Package Dev Handbook; see [[*References][References]] for that material and proper
attribution.

Buffer-name casing is deliberately /not/ legislated here; that decision is
owned by open bead =ogent-cnw.7=.

* File anatomy

- Every file's first line is the canonical header with lexical binding:
  =;;; ogent-models.el --- Model registry for ogent -*- lexical-binding: t; -*-=
  (=lisp/ogent-models.el:1=).
- Only the package entry point carries the full package header: Author,
  Version, Package-Requires, Homepage, Keywords, License (=lisp/ogent.el:1-25=).
  Sub-files NEVER repeat it.
- Sub-files use a slim layout: header line, =;;; Commentary:= with a short
  purpose paragraph, then =;;; Code:= (=lisp/ogent-models.el:3-10=).
- Every file ends with =(provide 'NAME)= followed by
  =;;; NAME.el ends here= (=lisp/ogent-edit-format.el:131-133=).

* Formatting & layout

- Standard Emacs Lisp indentation: two-space body indent, as produced by
  =indent-region= in =emacs-lisp-mode=.  The =lint-indent= rule of
  =make lint= enforces it mechanically (=Makefile:151=), so never
  hand-align against the grain of the indenter.
- Macros that take a body declare their indentation and edebug spec so
  callers indent like built-ins: =(declare (indent 1) (debug t))=
  (=test/ogent-armory-schedule-tests.el:19=).
- Major sections within a file are separated by =;;;= outline comments
  (=lisp/ogent-analytics.el:146=, =;;; Token Estimation=), which keeps
  =outline-minor-mode= navigation usable in large files.

* Naming

- Public symbols are =ogent-MODULE-name=:
  =ogent-analytics-estimate-tokens= (=lisp/ogent-analytics.el:148=).
- Private symbols are =ogent-MODULE--name= (double hyphen):
  =ob-ogent--expand-vars= (=lisp/ob-ogent.el:57=).
- Test-suite data and helpers use =ogent-MODULE-tests--NAME=:
  =ogent-edit-tests--backend= (=test/ogent-edit-tests.el:15=).
- No un-prefixed top-level symbols.  The single exception is
  =lisp/inline-diff.el= (=lisp/inline-diff.el:1=), a vendored standalone
  library that keeps its own =inline-diff-= prefix.

* Data structures

- Plists are the default shape for internal APIs: registry entries, context
  payloads, schedule events, conversation records.  Functions returning them
  say so: "Read CONVERSATION-ID from DIRECTORY and return a plist."
  (=lisp/ogent-armory-conversations.el:402-403=).
- Alists appear only at external-API boundaries where the caller dictates the
  shape, e.g. Org Babel header arguments
  (=org-babel-default-header-args:ogent=, =lisp/ob-ogent.el:53-55=) and the
  =:var= alist Babel passes in (=lisp/ob-ogent.el:59=).
- Stateful objects with identity use =cl-defstruct= with per-slot comments:
  =ogent-edit= (=lisp/ogent-edit-format.el:40=) and =ogent-ui-request=
  (=lisp/ui/ogent-ui.el:408=) are the canonical examples.

* Customization

- Exactly one =defgroup= per module, placed immediately after the requires at
  the top of =;;; Code:=, parented under =ogent=
  (=lisp/ogent-models.el:15-17=).
- Every =defcustom= carries both =:type= and =:group=, no exceptions
  (=lisp/ogent-models.el:19-22=).
- Defcustom docstrings open with a single summary sentence in the imperative
  / declarative style, then optional detail paragraphs
  (=lisp/ogent-models.el:53-58=).

* Autoloads, requires, declarations

- User-facing entry commands get an =;;;###autoload= cookie
  (=lisp/ogent-edit.el:542=).  Transient prefixes need the explicit form:
  =;;;###autoload (autoload 'ogent-edit-menu "ogent-edit" nil t)=
  (=lisp/ogent-edit.el:828=).
- The entry point requires modules in /dependency order/, not alphabetical
  order; ordering constraints are documented inline, e.g.
  =(require 'ogent-ui-theme)  ; Design system - load before UI=
  (=lisp/ogent.el:101-109=).
- Soft dependencies are declared, never required: pair
  =(declare-function f "module" ...)= with bare =(defvar var)= for the
  variables (=lisp/ob-ogent.el:34-38=).
- Symbols that =check-declare= cannot find (cl-defstruct accessors, macro- or
  runtime-generated functions) are declared with the arglist/fileonly flags
  =t t= and tagged with a =(fileonly: ...)= comment explaining why:
  =;; Struct accessors (fileonly: cl-defstruct-generated)=
  (=lisp/ogent-analytics.el:28-30=).

* Docstrings & checkdoc

- Docstrings open with an imperative verb ("Return...", "Estimate...",
  "Display...") and name arguments in caps:
  "Estimate token count for TEXT using character-based heuristic."
  (=lisp/ogent-analytics.el:149=).
- checkdoc cleanliness is a merge gate: =make lint= runs checkdoc,
  byte-compilation, check-declare, and indentation lints over =lisp/= and
  =test/= (=Makefile:127=, =AGENTS.md= "Build, Test, and Development
  Commands").

* Errors & messages

- User-correctable failures signal =user-error= with the offending value in
  the message, never plain =error=:
  =(user-error "Armory conversation not found: %s" file)=
  (=lisp/ogent-armory-conversations.el:298-299=).
- Echo-area messages from request machinery carry an =ogent:= prefix so they
  are attributable in =*Messages*=:
  =(message "ogent: %s completed (%s%s)" ...)= (=lisp/ogent-core.el:307=).
  Mode toggles announce themselves by mode name instead
  (=lisp/ogent-debug.el:108=).

* Buffer-local state

- Set buffer-local variables with =setq-local=, never
  =make-local-variable= + =setq= (=lisp/ogent-issues.el:569-571=).
- Variables that are always buffer-local are declared with =defvar-local=
  (=lisp/ogent-issues.el:1150=).

* Tests

- =test/= mirrors =lisp/= one-to-one: =lisp/ogent-armory-schedule.el= is
  covered by =test/ogent-armory-schedule-tests.el=; UI suites live under
  =test/ui/= mirroring =lisp/ui/=.
- Test names follow =ert-deftest ogent-MODULE-FEATURE-BEHAVIOR= and carry a
  one-sentence behavior docstring:
  =ogent-armory-schedule-expands-cron-and-heartbeats=
  (=test/ogent-armory-schedule-tests.el:50-51=).
- Shared Org fixtures live in =test/data/= and load through the helper:
  =(ogent-test-with-fixture "data/fixture.org" ...)=
  (=test/ogent-context-tests.el:10=).
- Stub collaborators with =cl-letf= on =symbol-function=; never redefine or
  advise inside tests (=test/ogent-issues-bd-tests.el:61-64=).  Stub only at
  the process/network boundary; assert observable behavior (returned plists,
  buffer text, files written), not private state, and never mock what can
  run for real.
- Filesystem tests use a per-suite temp-dir macro whose =unwind-protect=
  guarantees cleanup (=ogent-armory-schedule-test-with-temp-dir=,
  =test/ogent-armory-schedule-tests.el:17-24=).
