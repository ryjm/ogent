#+title: ogent

*An exoskeleton for Emacs, not an autonomous agent.*

ogent is an LLM integration layer for Org-mode that keeps you in control. It's built on the insight that there's a real, defensible layer between foundation models and end users—one that handles context engineering, multi-model orchestration, and application-specific UI so you can focus on verification, not prompt wrangling.

** The Exoskeleton Philosophy

AI generates instantly. But you're still responsible for the output. If you get a 1,000 line diff, you have to verify it actually works, introduces no bugs, and has no security issues. That takes time.

ogent addresses this with two levers:

1. *Speed up verification* — Visual diffs, structured Org blocks, keybindings for accept/reject. Your visual system is faster than reading chat transcripts.

2. *Keep the AI on a leash* — Explicit context via @handles, preview before sending, autonomy you control. Better to spend time on a precise prompt that succeeds on the first pass than spin in a re-prompt loop.

An exoskeleton amplifies your strength while you stay in control. An autonomous robot operates on its own. We're building the exoskeleton.

** What ogent does for you

- *Context engineering* — You don't copy-paste files and error logs. ogent curates what goes into the context window via @handles, codemap, and pinned items.
- *Multi-call orchestration* — Fan out to multiple models, compare responses side-by-side, apply the best one.
- *Org-native UI* — Responses land in structured blocks you can fold, edit, and version. Red/green diffs for code changes. =C-c C-c= to accept.
- *Autonomy slider* — Quick Q&A (=C-c . ?=) for small queries. Full context dispatch (=C-c . p=) for deeper work. You control the scope.

* Requirements
- Emacs 29.1 or newer with Org-mode
- [[https://github.com/karthink/gptel][gptel]] for transport and model selection
- Optional: [[https://www.orgroam.com/][org-roam]] for cross-file handle resolution
- An LLM provider:
  - **Claude Max/Pro**: Use OAuth authentication (recommended, no API key needed)
  - **API Key**: Traditional API key for Anthropic or OpenAI

* Install
1. Clone the repository:
   #+begin_src shell
   git clone https://github.com/<you>/ogent.git
   cd ogent
   #+end_src
2. Ensure dependencies compile cleanly:
   #+begin_src shell
   make lint
   make test
   #+end_src
3. Add =lisp/= and =lisp/ui/= to your =load-path=, then require =ogent=:  
   #+begin_src emacs-lisp
   (add-to-list 'load-path (expand-file-name "lisp" "/path/to/ogent"))
   (add-to-list 'load-path (expand-file-name "lisp/ui" "/path/to/ogent"))
   (require 'ogent)
   #+end_src

* Configuration
** Quick Setup (Recommended)
Run =M-x ogent-onboard= to interactively configure your LLM provider. This wizard:
1. Lets you choose between Claude Max/Pro (OAuth) or API key authentication
2. Guides you through the login process
3. Configures gptel backends automatically

** Claude Max/Pro OAuth (No API Key Required!)
If you have a Claude Pro or Claude Max subscription, you can authenticate via OAuth:

#+begin_src emacs-lisp
(require 'ogent)
(ogent-anthropic-login)  ; Opens browser, paste the code
#+end_src

This stores refresh tokens in =~/.emacs.d/ogent/anthropic-oauth/= and automatically
refreshes them as needed. No API key management required!

To check status: =M-x ogent-anthropic-status=
To logout: =M-x ogent-anthropic-logout=

** Vanilla Emacs (API Key)
#+begin_src emacs-lisp
(add-hook 'org-mode-hook #'ogent-mode)
(setq ogent-default-model "gpt-4o-mini")
#+end_src

** Doom Emacs
#+begin_src emacs-lisp
(package! ogent :recipe (:local-repo "/path/to/ogent"))
(use-package! ogent
  :after org
  :config
  (map! :leader :desc "Ogent Prompt" "o p" #'ogent-prompt-dispatch)
  (ogent-global-mode 1))
#+end_src

* Usage
- Open any Org buffer and toggle =ogent-mode= (or rely on =ogent-global-mode=).
- Press =C-c . p= to open the dispatcher, pick models, and send the current subtree.
- Use =C-c . c= for a context preview and =C-c . m= to inspect the codemap.
- Press =C-c . d= to capture the last AI response to a Notes child heading.
- Use =C-c . ?= for quick Q&A without full context machinery.
- To attach reusable prompt snippets, create Org nodes with an =OGENT_ID= (see =docs/prompts.org=).
