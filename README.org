#+title: ogent

*LLMs as Unix tools, not chatbots.*

ogent brings language models into Org-mode the Emacs way: composable commands, text as the universal interface, and you in control. No chat windows. No copy-pasting between apps. Just structured context in, structured responses out.

** Why an exoskeleton, not an agent

The model generates text instantly. You're the one who has to verify it works. That's the bottleneck—not generation, but review.

ogent is designed around this constraint:

- *Make review fast* — Responses land in Org blocks you can fold, diff, and navigate with standard keybindings. Visual diffs beat reading prose. =C-c C-c= beats typing "yes".

- *Make context explicit* — No magic "send my whole project". You declare what goes in via =@handles=, pins, and codemap. Precise input means predictable output.

The model is a tool. You wield it. That's the difference between an exoskeleton and a robot.

** What ogent provides

- *Context as data* — =@handles= resolve to Org nodes, files, or org-roam entries. Codemap indexes your project. Pins persist across requests. You see exactly what the model sees before sending.

- *Responses as structure* — Output lands in src blocks with metadata (=:model=, =:status=, =:latency=). Fold them, edit them, version them. It's just Org.

- *Edits as diffs* — Code changes render as smerge conflicts. Accept with =C-c C-c=, reject with =C-c C-k=. No ambiguity about what changed.

- *Commands, not conversations* — =C-c . ?= for quick questions. =C-c . p= for full context dispatch. =C-c . E= for code edits. Each does one thing.

* Requirements
- Emacs 29.1 or newer with Org-mode
- [[https://github.com/karthink/gptel][gptel]] for transport and model selection
- Optional: [[https://www.orgroam.com/][org-roam]] for cross-file handle resolution
- An LLM provider:
  - **Claude Max/Pro**: Use OAuth authentication (recommended, no API key needed)
  - **API Key**: Traditional API key for Anthropic or OpenAI

* Install
1. Clone the repository:
   #+begin_src shell
   git clone https://github.com/<you>/ogent.git
   cd ogent
   #+end_src
2. Ensure dependencies compile cleanly:
   #+begin_src shell
   make lint
   make test
   #+end_src
3. Add =lisp/= and =lisp/ui/= to your =load-path=, then require =ogent=:  
   #+begin_src emacs-lisp
   (add-to-list 'load-path (expand-file-name "lisp" "/path/to/ogent"))
   (add-to-list 'load-path (expand-file-name "lisp/ui" "/path/to/ogent"))
   (require 'ogent)
   #+end_src

* Configuration
** Quick Setup (Recommended)
Run =M-x ogent-onboard= to interactively configure your LLM provider. This wizard:
1. Lets you choose between Claude Max/Pro (OAuth) or API key authentication
2. Guides you through the login process
3. Configures gptel backends automatically

** Claude Max/Pro OAuth (No API Key Required!)
If you have a Claude Pro or Claude Max subscription, you can authenticate via OAuth:

#+begin_src emacs-lisp
(require 'ogent)
(ogent-anthropic-login)  ; Opens browser, paste the code
#+end_src

This stores refresh tokens in =~/.emacs.d/ogent/anthropic-oauth/= and automatically
refreshes them as needed. No API key management required!

To check status: =M-x ogent-anthropic-status=
To logout: =M-x ogent-anthropic-logout=

** Vanilla Emacs (API Key)
#+begin_src emacs-lisp
(add-hook 'org-mode-hook #'ogent-mode)
(setq ogent-default-model "gpt-4o-mini")
#+end_src

** Doom Emacs
#+begin_src emacs-lisp
(package! ogent :recipe (:local-repo "/path/to/ogent"))
(use-package! ogent
  :after org
  :config
  (map! :leader :desc "Ogent Prompt" "o p" #'ogent-prompt-dispatch)
  (ogent-global-mode 1))
#+end_src

* Usage
- Open any Org buffer and toggle =ogent-mode= (or rely on =ogent-global-mode=).
- Press =C-c . p= to open the dispatcher, pick models, and send the current subtree.
- Use =C-c . c= for a context preview and =C-c . m= to inspect the codemap.
- Press =C-c . d= to capture the last AI response to a Notes child heading.
- Use =C-c . ?= for quick Q&A without full context machinery.
- To attach reusable prompt snippets, create Org nodes with an =OGENT_ID= (see =docs/prompts.org=).

|-----------+-------------------------------+------------------------------------------|
| =C-c . p= | =ogent-prompt-dispatch=       | Open dispatcher to select models/presets |
| =C-c . r= | =ogent-request=               | Send request with current context        |
| =C-c . a= | =ogent-abort-request=         | Abort current request                    |
| =C-c . R= | =ogent-retry-request=         | Retry last request                       |
| =C-c . c= | =ogent-context-preview=       | Preview context before sending           |
| =C-c . m= | =ogent-codemap-buffer=        | Generate codemap for current buffer      |
| =C-c . P= | =ogent-pin-dwim=              | Pin file/buffer/region to context        |
| =C-c . U= | =ogent-unpin-interactive=     | Unpin item from context                  |
| =C-c . l= | =ogent-list-pinned=           | List all pinned context items            |
| =C-c . e= | =ogent-edit-menu=             | Edit menu for inline code changes        |
| =C-c . E= | =ogent-request-edit=          | Request code edits for buffer            |
| =C-c . s= | =ogent-edit-goto-source=      | Go to source from companion              |
| =C-c . C= | =ogent-edit-goto-companion=   | Go to companion from source              |
| =C-c . t= | =ogent-debug-tools-menu=      | Tools debug menu                         |
| =C-c . T= | =ogent-tool-rerun=            | Re-run tool at point                     |
| =C-c . b= | =ogent-show-backlinks=        | Show backlinks to current heading        |
| =C-c . g= | =ogent-show-dependency-graph= | Show dependency graph                    |
| =C-c . o= | =ogent-open-block=            | Open source block with ogent-mode        |
| =C-c . i= | =ogent-issues=                | Open issue tracker                       |
| =C-c . S= | =ogent-session-save=          | Save current session                     |
| =C-c . L= | =ogent-session-load=          | Load a saved session                     |
| =C-c . H= | =ogent-session-list=          | List saved sessions                      |
