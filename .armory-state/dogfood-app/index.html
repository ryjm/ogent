Findings

Ordered by the run's severity labels and written from the session output.

P2

Two competing primary surfaces

Problem
ogent-armory-status and ogent-armory-home both read as the main entry point, depending on which doc a contributor opens first.
Impact
New work can land on the wrong surface and deepen product confusion.
Fix direction
Choose one canonical entry point and align the rest, with Home-first behavior looking like the cleanest fit.
P2

Storage contract drift

Problem
The scaffold advertises .agents/.conversations and .jobs, while live session traffic actually uses per-agent sessions/.
Impact
Future work can split state across documented and undocumented paths with no early warning.
Fix direction
Delete the unused story or wire it up for real, then document one canonical transcript location.
P2

Jobs are first-class on paper, hidden in practice

Problem
Docs and onboarding point users at ogent-armory-jobs, but Home and Status do not advertise that surface.
Impact
The first-run workflow has a UX seam right where Armory is supposed to feel operationally clear.
Fix direction
Expose Jobs from Home or Status, or rewrite the onboarding path around the real navigation model.
P3

Evil support adds a maintenance sync point

Problem
Mode registration and overriding-map setup are duplicated across Armory status and richer UI modules.
Impact
Low immediate risk, steady future tax every time another Armory mode appears.
Fix direction
Centralize Evil registration behind one helper or one shared mode list.
